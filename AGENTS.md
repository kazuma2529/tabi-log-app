# AGENTS.md

このリポジトリで開発するエージェントは、必ず以下のルールに従うこと。

## 開発前に必ず確認するもの

実装を始める前に、必ず以下を確認する。

1. `docs/requirements.md`
2. `docs/plan.md`
3. `images/app-image.png`
4. `images/app-image2.png`

`images/` フォルダの2枚の画像は、このアプリの最終的な完成イメージである。毎回この画像を確認し、「このようなアプリを作る」というゴールからずれないように開発すること。

## plan.md に沿って進める

- 開発時は必ず `docs/plan.md` を見て、現在どのフェーズを進めているか確認する。
- 原則として `docs/plan.md` の順番に沿って実装する。
- 大きな工程が完了したら、`docs/plan.md` のチェックリストを更新する。
- 計画から外れる必要がある場合は、理由を明確にしてから進める。

## 参考画像の扱い

参考画像は単なる飾りではなく、UI のゴールである。

特に以下を意識する。

- 旅ノート、紙、地図、スタンプ、手帳のような温かい世界観
- teal 系と gold/brown 系のアクセント
- 下部タブ中央の丸いプラスボタン
- ホーム、地図、日記、統計の4タブ構成
- 余白があり、カードが読みやすい画面
- 訪問記録追加フローの分かりやすさ

## RevenueCat と App Store 対応

- RevenueCat 連携、App Store Connect、EAS Build、本番課金、App Store 申請は開発の後半で対応する。
- 初期段階では RevenueCat を組み込まない。
- まずはローカル保存と主要画面を作り、課金状態は開発用の仮フラグで扱う。
- 写真無制限と年別分析の有料制御は、後で RevenueCat に差し替えやすい構造にしておく。

## 自律的な確認

実装後は、可能な範囲でエージェントが自律的に確認する。

- 型チェック
- lint
- テスト
- アプリ起動
- 画面表示確認
- スクリーンショット確認
- 参考画像との見た目の比較
- 主要フローの操作確認

確認できないことがある場合は、できなかった理由と、次に必要な確認方法を明記する。

## 実装方針

- iOS 専用アプリとして考える。
- Expo SDK 54、Expo Router、Expo SQLite を前提にする。
- バックエンド、認証、iCloud 同期は作らない。
- データは端末ローカル保存にする。
- 旅行計画アプリではなく、人生で訪れた国を記録する世界制覇ログアプリとして作る。
- 迷った場合は、機能追加よりも「記録しやすさ」と「完成イメージに近い体験」を優先する。
