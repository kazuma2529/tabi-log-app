# 実装計画書：世界旅記録アプリ

この計画書は、`docs/requirements.md` の要件と `images/` フォルダ内の完成イメージをもとに、開発初心者でも着手しやすい順番で実装を進めるためのものです。

RevenueCat、App Store 申請、EAS Build、本番課金設定などの初めてだと詰まりやすい作業は、アプリ本体の主要機能が動いてから最後の方で対応します。

---

## 0. 開発時の前提ルール

- 開発を始める前に必ず `docs/requirements.md` とこの `docs/plan.md` を確認する。
- 開発を始める前に必ず以下の完成イメージを確認し、見た目の方向性からずれないようにする。
  - `images/app-image.png`
  - `images/app-image2.png`
- 旅ノート、紙、地図、スタンプ、手帳のような温かい世界観を維持する。
- 主なアクセントカラーは、完成イメージに近い teal 系と gold/brown 系を基本にする。
- 下部タブ中央の丸いプラスボタンは、アプリの主要導線として早い段階で再現する。
- 実装後は、可能な限り Codex 側で自律的に動作確認、スクリーンショット確認、表示崩れ確認を行う。
- `docs/plan.md` は進捗管理にも使う。大きな工程が完了したらチェックを更新する。

## 1. 開発方針

### 1.1 最初に作るもの

最初は「課金なし」「App Store 申請なし」「RevenueCat なし」で、ローカルだけで動くアプリを作る。

優先するもの：

- Expo アプリの土台
- 画面遷移
- 下部タブと中央プラスボタン
- 国マスタ
- SQLite 保存
- 訪問記録追加
- ホーム、地図、日記、統計、バケットリスト
- 参考画像に近いデザイン

後回しにするもの：

- RevenueCat 連携
- App Store Connect 設定
- 実機課金テスト
- TestFlight
- App Store 申請
- iPad 最適化
- データエクスポート
- ウィジェット

### 1.2 技術方針

- Expo SDK 54 を前提にする。
- iOS 専用アプリとして設計する。
- ルーティングは Expo Router を使う。
- データ保存は Expo SQLite を使う。
- 写真はアプリ内ストレージに保存し、SQLite にはファイルパスを保存する。
- 課金状態は最初は開発用の仮フラグで扱い、最後に RevenueCat へ差し替える。
- RevenueCat が必要になるまでは Expo Go で確認できる範囲を優先する。

## Phase 4：SQLite の導入とデータ保存

目的：訪問記録、都市、写真、メモ、バケットリストを端末内に保存できるようにする。

- [x] Expo SQLite を導入する。
- [x] DB 初期化処理を作る。
- [x] マイグレーションの仕組みを作る。
- [x] `countries`、`visits`、`cities`、`photos`、`memo_cards`、`bucket_list`、`purchases` のテーブルを作る。
- [x] DB 操作用の関数を `src/db/` にまとめる。
- [x] 訪問回数、最後に行った日付、世界制覇率などを算出する query/helper を作る。
- [x] 仮データ表示を SQLite 読み込みへ切り替える。

重要な仕様：

- 訪問回数は国ごとの `visits` 件数から算出する。
- `visit_order` は保存時点の `同じ country_id の visits 件数 + 1` とする。
- 表示順は `visited_at` の古い順を基本にする。
- バケットリストは同じ国を重複登録できない。
- バケットリスト登録国を訪問済みにした場合は、自動でバケットリストから削除する。

完了条件：

- アプリを再起動しても登録データが残る。
- バケットリストの追加、削除、重複防止が動く。

## 3. 自律的な確認方針

Codex が実装を進める際は、可能な限り以下を自律的に行う。

- 型チェックを実行する。
- lint を実行する。
- テストがある場合はテストを実行する。
- アプリを起動できる場合は起動する。
- iOS シミュレーターや利用可能な確認手段で画面を開く。
- 主要画面のスクリーンショットを撮る。
- `images/app-image.png` と `images/app-image2.png` の方向性と比べる。
- 文字のはみ出し、ボタンの押しにくさ、画面下部の隠れ、タブバーとの重なりを確認する。
- 訪問記録の追加、バケットリスト追加、削除、国詳細表示などの主要フローを実際に操作して確認する。

できない場合は、何ができなかったのかを記録し、次に必要な確認方法を明確にする。

## 4. MVP 完了条件

MVP は以下が動いた時点で完了とする。

- ホームタブが表示される。
- 地図タブが表示される。
- 日記タブが表示される。
- 統計タブが表示される。
- 中央プラスボタンから訪問記録を追加できる。
- 国、訪問日、都市、写真、メモを保存できる。
- 同じ国に複数回の訪問記録を保存できる。
- 国詳細で訪問回タブを切り替えられる。
- バケットリストを追加、表示、削除できる。
- SQLite に保存され、アプリ再起動後もデータが残る。
- 無料版の写真10枚制限が動く。
- 年別分析が有料版限定としてロックされる。
- 参考画像の方向性に近い見た目になっている。

## 5. 開発メモ

- このアプリは旅行計画アプリではなく、人生で訪れた国を記録する世界制覇ログアプリである。
- 便利機能を増やしすぎるより、記録したくなる見た目と、迷わず登録できる体験を優先する。
- 初心者が後で対応しやすいように、RevenueCat と App Store まわりは最後にまとめて扱う。
- 迷ったときは `docs/requirements.md` の MVP 範囲と受け入れ基準に戻る。
